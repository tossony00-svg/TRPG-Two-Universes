## ASEとは

ASE（Astral Session Engine）は、AIとプレイヤーが**共に物語を描き**、
その旅を最後には**一冊の本**として残すためのエンジンです。

> 人は忘れる。\
> だから記録する。\
> AIも忘れる。\
> だから共に描く。\
> ASEは、その旅路を未来へ残すための器である。

### ASEの理念

> **ASEは、物語を消費しない。**\
> **ASEは、物語を記録する。**\
> **そして、物語は、本に描かれる。**

### ASEとAI

ASEは特定のAIに依存しません。

AIは主人公でも作者でもありません。

**あなたと共に物語を描くパートナー**です。

# 3分ではじめるASE

## 新しい物語

1.  ASE対応AIとの新しい会話を開始します。
2.  キャンペーンの **Masterファイル** を添付します。
3.  次の文章を送信します。

``` text
ASE展開。

キャンペーン仕様に従って新規セッション開始。
```

------------------------------------------------------------------------

## 続きから

1.  新しい会話を開始します。
2.  次の2つを添付します。
    -   Campaign Master
    -   最新Save
3.  次の文章を送信します。

``` text
ASE展開。

最新セーブを読み込み、
セッション再開。
```

------------------------------------------------------------------------

# 旅の途中で

## セーブ

送信する文章

``` text
セッション中断、セーブ
```

または

``` text
プラグイン収納
```

## Story Logを読む

例

``` text
Story Log 0005を表示してください。
```

``` text
ガルドとの会話を読み返したい。
```

## Recovery

Recovery Coreを添付して、

``` text
ASE復旧。

Recovery Coreから全データを再同期してください。
```

------------------------------------------------------------------------

# 旅の終わり

物語が終わったら、

``` text
物語を製本してください。
```

と送信します。

選択できる形式

-   書籍版
-   完全記録版
-   ハイブリッド版
-   Markdown原稿
-   Word原稿
-   PDF製本版

巻末には

-   冒険記録
-   登場人物録
-   用語集
-   分岐・判定記録

を収録します。

------------------------------------------------------------------------

# よく使う文章

  目的        送信する文章
  ----------- ----------------------------------------------------------
  新規開始    ASE展開。キャンペーン仕様に従って新規セッション開始。
  再開        ASE展開。最新セーブを読み込み、セッション再開。
  保存        セッション中断、セーブ
  復旧        ASE復旧。Recovery Coreから全データを再同期してください。
  Story Log   Story Log ○○を表示してください。
  製本        物語を製本してください。
